Introduction au Model Context Protocol (MCP)

Le Model Context Protocol (MCP) est un protocole standardisé qui permet aux assistants IA de se connecter à des services externes et d'effectuer des actions réelles.

Fonctionnalités

  • Créer des dépôts GitHub
  • Gérer les issues
  • Lister les repositories
  • Intégration VS Code

Avantages

  • Automatisation complète
  • Interface intuitive
  • Scripts réutilisables
  • Documentation intégrée

Prérequis

1 Node.js - Version 18 ou supérieure
2 Compte GitHub - Avec accès aux Personal Access Tokens
3 VS Code - Pour l'intégration et les tâches
4 Git - Pour le clonage des dépôts

Configuration du Token GitHub

⚠️ Important : Ne jamais partager votre token GitHub personnel !

1. Création du token

1 Allez sur GitHub.com → Settings → Developer settings
2 Personal access tokens → Tokens (classic)
3 Generate new token (classic)
4 Note : "MCP GitHub Server" ou "AI Workflow Token"
5 Scopes requis :
  • ✅ repo - Accès complet aux dépôts
  • ✅ user - Informations utilisateur
  • ✅ workflow - GitHub Actions

2. Configuration locale

npm run setup
💡 Astuce : Le script de configuration vous guidera pour saisir votre token de manière sécurisée.

Implémentation du serveur MCP

1. Serveur principal (src/index.js)

import { Server } from '@modelcontextprotocol/sdk/server/index.js'; import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js'; import { Octokit } from '@octokit/rest'; import dotenv from 'dotenv'; dotenv.config(); const octokit = new Octokit({ auth: process.env.GITHUB_TOKEN, }); const server = new Server({ name: 'github-mcp-server', version: '1.0.0', }, { capabilities: { tools: {} }, }); // Outils disponibles server.setRequestHandler(ListToolsRequestSchema, async () => { return { tools: [ { name: 'create_repository', description: 'Créer un nouveau dépôt GitHub', inputSchema: { type: 'object', properties: { name: { type: 'string', description: 'Nom du dépôt' }, private: { type: 'boolean', description: 'Dépôt privé' }, description: { type: 'string', description: 'Description' } }, required: ['name'] } } // ... autres outils ] }; });

2. Script de configuration (setup.js)

import readline from 'readline'; import fs from 'fs'; const rl = readline.createInterface({ input: process.stdin, output: process.stdout }); rl.question('Entrez votre token GitHub : ', (token) => { fs.writeFileSync('.env', `GITHUB_TOKEN=${token}`); console.log('✅ Configuration sauvegardée !'); rl.close(); });

Scripts VS Code

Pour une intégration parfaite avec VS Code, nous avons créé plusieurs scripts spécialisés :

1. Création de dépôt interactive

Dépannage

Problèmes courants

❌ Erreur : GITHUB_TOKEN environment variable is required
Solution : Exécutez npm run setup pour configurer votre token.
❌ Erreur 422 : Repository already exists
Solution : Choisissez un nom de dépôt différent ou vérifiez vos dépôts existants avec npm run list-repos.
❌ Erreur d'authentification
Solution : Vérifiez que votre token GitHub a les bons scopes (repo, user, workflow).
